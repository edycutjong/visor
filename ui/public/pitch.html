Slide 04 // The Solution

Visor — The Privacy-Blind Gateway

Visor keeps the coordinator completely blind. Users encrypt their PII locally. Unsecure agents schedule actions using secure placeholders (e.g. `{{profile.ssn}}`). Decryption and placeholder injection occur exclusively within a secure TEE enclave during out-of-band egress.

Slide 06 // Cryptographic Moat

Groth16 ZK Proofs + ECIES

To prevent malicious actors from submitting form actions on behalf of other users, Visor requires a Groth16 ZK-SNARK proof. The Rust TEE contract verifies that the user owns the email hash commitment offline without revealing any identity parameters.

Slide 07 // x402 Micropayments

DDoS Prevention via x402

To prevent spam and DDoS attacks on target API endpoints, Visor integrates x402 micropayments. Every outbound submission requires payment token verification before the enclave dispatches the request, protecting the egress gateway from abuse.
