# Visor — Demo Protocol

This guide walks through the step-by-step demo protocol for judges to reproduce and verify **Visor** functionality.

---

## 1. Setup & Environment
- **Prerequisites:** Node.js ≥ 20.9.0, Rust, and the Terminal 3 Local Sandbox CLI installed.
- **Run Seeding:**
  ```bash
  python3 scripts/seed.py
  ```
  *This registers the `clinic-intake` template and seeds the profile context for `did:t3n:maria123`.*

---

## 2. Step-by-Step Walkthrough

### Step 1: Initialize User Profile & Delegation
1. Open the UI at `http://localhost:3000`.
2. Select **Onboard User** to view Maria's local mock profile (first name: `Maria`, DOB: `1994-08-14`, email: `maria.s@healthmail.net`).
3. Click **Authorize Agent** to grant the Visor agent delegation scope. Under the hood, this executes the T3 `agent-auth` grant registering the allowed functions (`blind-submit`, `finalize`) and target hosts (`https://clinic.sandbox.test`).

### Step 2: Request Action
1. In the chat interface, input:
   > *"Book a dermatologist consult for me this week."*
2. The agent's LLM matches the intent with the `clinic-intake` template and executes:
   ```json
   {
     "templateId": "clinic-intake",
     "plan": {
       "symptom": "dermatology consult",
       "preferredWindow": "this week"
     }
   }
   ```

### Step 3: Split-Screen Egress Verification
1. As the request executes, watch the split-screen console reveal:
   - **Left Panel (Agent Context):** The outgoing body payload generated by the Rust WASM contract:
     ```json
     {
       "first_name": "{{profile.first_name}}",
       "dob": "{{profile.date_of_birth}}",
       "email": "{{profile.verified_contacts.email.value}}",
       "symptom": "dermatology consult",
       "preferredWindow": "this week"
     }
     ```
   - **Right Panel (Egress Wire):** The actual HTTP packet payload intercepted on the target clinic API side:
     ```json
     {
       "first_name": "Maria",
       "dob": "1994-08-14",
       "email": "maria.s@healthmail.net",
       "symptom": "dermatology consult",
       "preferredWindow": "this week"
     }
     ```

### Step 4: Audit & Verification
1. Click **Verify Receipt** in the dashboard.
2. The UI checks the cryptographic signature of the Verifiable Credential against the DID Registry.
3. Observe the green badge showing: `Signature Verified. Signer DID: did:t3n:visor-signer-key`.
4. Inspect the immutable audit log showing:
   - Action: `blind-submit`
   - Target: `https://clinic.sandbox.test/intake`
   - Markers Used: `[first_name, date_of_birth, email]`
   - Plaintext leaked: `None`
